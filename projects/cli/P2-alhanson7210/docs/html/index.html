About This Project:

The outermost layer of the operating system kernel is called the shell. In Unix-based systems, the shell is generally a command line interface. Most Linux distributions ship with bash as the default (there are several others: csh, ksh, sh, tcsh, zsh). In this project, what will be implementing is a shell.

Program options:

Upon startup, the shell will print its prompt and wait for user input. The shell should be able to run commands in both the current directory and those in the
PATH environment variable. Execvp was used to do this. To run a command in the current directory, you’ll need to prefix it with ./ as usual. If a command isn’t found, it prints an error message:

To compile and run:

make
./flash
